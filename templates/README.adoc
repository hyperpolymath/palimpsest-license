= SCM File Templates
:toc:

== Overview

These templates provide starting points for machine-readable project metadata files.
Copy them to `.machine_readable/` in your project and fill in your project's details.

== Files

[cols="1,2"]
|===
| Template | Purpose

| `STATE.scm.template`
| Track current project state, progress, blockers, and session history

| `ECOSYSTEM.scm.template`
| Define project's position in the ecosystem and relationships

| `META.scm.template`
| Document architecture decisions and development practices
|===

== Usage

[source,bash]
----
# Create the directory
mkdir -p .machine_readable

# Copy and rename templates
cp templates/STATE.scm.template .machine_readable/STATE.scm
cp templates/ECOSYSTEM.scm.template .machine_readable/ECOSYSTEM.scm
cp templates/META.scm.template .machine_readable/META.scm

# Edit each file to reflect your project
----

== Schema

All files use Guile Scheme S-expression format with specific media types:

- `STATE.scm` → `application/vnd.state+scm`
- `ECOSYSTEM.scm` → `application/vnd.ecosystem+scm`
- `META.scm` → `application/meta+scheme`
