= Documentation Style Guide – Palimpsest‑MPL
:toc: macro

This guide defines the conventions for writing and maintaining documentation across the Palimpsest‑MPL ecosystem.

toc::[]

== 1. Format

All narrative and conceptual documents must use **AsciiDoc**.  
Exceptions:

* GitHub Issue Templates → Markdown  
* GitHub PR Templates → Markdown  
* GitHub Discussions Templates → Markdown  

== 2. Tone and Voice

Documentation must be:

* clear  
* narratively coherent  
* provenance‑aware  
* agent‑friendly  
* neutral and precise  

Avoid marketing language.  
Prefer conceptual clarity over verbosity.

== 3. Structure

Use:

* `= Title`  
* `== Section`  
* `=== Subsection`  

Always include:

----
// SPDX-License-Identifier: PMPL-1.0-or-later
----


== 4. Linking

Use `xref:` for internal links:

----
xref:../governance/GOVERNANCE.adoc[Governance Model]
----


Use `link:` for external URLs.

== 5. Provenance Notes

Major documents should include:

* origin  
* version  
* related exhibits  
* governance status  

== 6. Diagrams

Prefer:

* ASCII diagrams  
* PlantUML blocks  
* AsciiDoc tables  

== 7. File Naming

Use:

* lowercase  
* hyphens  
* `.adoc` extension  

Examples:

* `journey-map.adoc`  
* `provenance-flow-diagram.adoc`  
* `tooling.adoc`  

== 8. Review

Documentation changes must follow:

* Maintainer Review Checklist  
* Governance rules  
* Versioning guidelines
